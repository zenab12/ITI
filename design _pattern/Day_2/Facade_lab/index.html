Facade design pattern

The facade design pattern is a structural design pattern that provides a simplified interface to a library, a framework, or any other complex set of classes.

Problem

Imagine that you are working on a project that uses a complex set of classes. You are working on a part of the project that uses a lot of these classes. You are not interested in the details of how these classes work. You just want to use them to get the job done. You don't want to have to learn how to use all of these classes. Solution

The facade design pattern provides a simplified interface to a complex subsystem of classes, library or framework. A facade can:

  • Make a software library easier to use, understand and test, since the facade has convenient methods for common tasks;
  • Make the library more readable, for the same reason;
  • Reduce dependencies of outside code on the inner workings of a library, since most code uses the facade, thus allowing more flexibility in developing the system;
  • Wrap a poorly designed collection of APIs with a single well-designed API.
